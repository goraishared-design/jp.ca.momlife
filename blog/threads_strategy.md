## スクリーンショットから分かった傾向

| コンテンツ種別 | 実績例 | リーチ・エンゲージメント |
|---|---|---|
| 業界インサイドの一言リプライ | 「航空会社2社で運賃担当していました！」 | 表示1万回・59いいね ⭐️最高 |
| 驚きの豆知識 | 「カナダだとNetflixでジブリ見られる」 | 11コメント・10いいね |
| 日常・共感系 | 「子供の看病。三種の神器」 | 6いいね・ほっこり系 |
| 時事反応 | WestJet/Air Canada 25%オフクーポン | 速報性あり |

→ **「業界の内側の人」として反応・コメントすること**が最もリーチが取れる。
→ Routineは「スケジュール投稿」を担当し、時事・リプライは引き続き手動で行う。

## 投稿の5カテゴリ

### ① IG連動Threads（Routine担当）

- **タイミング**: IGカルーセル投稿日（月・水・土）の同日 JST 21:30（IG投稿の30分後）
- **内容**: カルーセルの核心を1スレッドに圧縮。「インスタに詳しくまとめました」でIG誘導
- **URL**: なし（IGへの誘導はプロフリンク案内）
- **形式**: 150〜300字。カジュアルな話し言葉。うりの実感・感情を入れる

### ② ブログ連動Threads（Routine担当）

- **タイミング**: ブログ公開後、翌日 JST 12:00（ランチタイム）
- **内容**: 記事の「一番驚く事実」1点だけを冒頭に置き、URLで誘導
- **URL**: ブログ記事のURL（Blotatoのtextに直接記載）→ Threadsで自動リンク化
- **形式**: 冒頭フック → 2〜3行の補足 → URL → ハッシュタグ3〜4個

### ③ 時事ネタThreads（Routine担当）

- **タイミング**: 火・木の発火時にWeb検索で関連ニュースを確認 → あれば JST 20:00 に予約
- **ソース優先順**: Bloomberg Education / NYT / 文科省 / 航空業界ニュース / Statistics Canada
- **テーマ**: 教育・AI・キャリア・カナダ移住・為替・航空券（これはうり専門領域）
- **スタンス**: うりの視点で「これ、分かる」「これ、業界の中にいた私から言うと」
- **形式**: ニュースの要約1〜2行 → うりのひとこと視点 → 出典URL or ハッシュタグ

### ④ 単発日常Threads（Routine担当・週1程度）

- **タイミング**: 土曜 JST 18:00（週末の生活感）
- **内容**: カルガリーの季節・子育ての瞬間・二拠点あるある
- **形式**: 短め（100字程度）。オチか共感ポイントで終わる

### ⑤ 業界インサイドリプライ（手動・うり判断）

- Routineは担当しない。うりが他者の投稿にリプライするもの
- 「元航空会社Pricing Analyst」「元外資系アナリスト」の視点で反応
- これが最もバズりやすい。積極的に活用を

## ブログURLの紐づけ方法

Routineが発火したとき、以下のファイルを参照してURLを紐づける：

```
https://raw.githubusercontent.com/goraishared-design/jp.ca.momlife/main/blog/published_articles.md
```

このファイルには公開済みブログ記事のURL一覧を記載。Routineは投稿テーマと照合し、関連URLがあれば本文に挿入する。

## Threadsの文体ルール

- 話し言葉で書く（「です・ます」より「〜んですよね」「〜だった」）
- 重たい句読点は使わない（。の多用を避ける）
- 改行を多用して読みやすく
- 感情・驚き・共感を冒頭に置く
- 長くても300字。短い方が読まれる
- 絵文字は1〜3個まで（過剰にしない）
- うりの声：Xのアーカイブ1,100ポスト準拠

## Routine の注意点

- Threadsはテキストのみ投稿でOK（画像は任意）
- 1投稿500文字まで（Threads制限）
- IG連動の場合は、同じ内容をそのまま転載しない → 「別の切り口」で書き直す
- 公開されていないブログURLは絶対に使わない（published_articles.mdに掲載済みのものだけ）
