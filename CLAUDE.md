## IG × ブログ 整合性チェックルール

**plan.md（IGスケジュール）またはplanned_topics.md（ブログトピック）のどちらかを変更したとき、必ず両ファイルを照合して以下を確認・修正する。**

### チェック対象の対応表

| plan.md の項目 | planned_topics.md の対応項目 |
|---|---|
| 各投稿の「タイトル案」 | 連動トピックの「タイトル案」 |
| 各投稿の「日付」 | 連動トピックの「連動IG」日付 |
| 各投稿の「status」 | 連動トピックの「status」（投稿済→ブログも確認） |
| 各投稿の「連動ブログ」 | 対応トピックの存在確認 |

### 整合性チェックの手順

1. **plan.mdを変更した場合**（投稿日・タイトル・ピラーの変更）
   - planned_topics.mdを読み込む
   - 変更した投稿に紐づくトピックの「連動IG」日付・タイトルが一致しているか確認
   - ズレがあれば planned_topics.md を最新のplan.mdに合わせて修正

2. **planned_topics.mdを変更した場合**（タイトル変更・日程変更）
   - plan.mdを読み込む
   - 変更したトピックに紐づくIG投稿の「連動ブログ」欄と一致しているか確認
   - ズレがあれば plan.md を最新の planned_topics.md に合わせて修正

3. **どちらを正とするか（コンフリクト時のルール）**
   - うりが直接編集した方を「最新の決定」とする
   - 判断できない場合はSlackの `#sns投稿確認` に確認を投げてから修正する：
   ```
   ⚠️ *ファイル整合性の確認が必要です*
   plan.md と planned_topics.md で以下の項目がズレています：
   - [項目名]：plan.md → [値A] / planned_topics.md → [値B]
   どちらが正しいですか？
   ```

4. **両ファイルを同時に更新する**
   - GitHub APIでplan.mdのSHAを取得→更新
   - 続けてplanned_topics.mdのSHAを取得→更新
   - 片方だけ更新して終わらない

## A｜IGカルーセル下書き（公開前日 JST 22:00 に実行）

### A-1: plan.md を読む
以下のURLから**毎回必ず新規に**取得する（キャッシュ・前回セッションの記憶は使わない）：
```
https://raw.githubusercontent.com/goraishared-design/jp.ca.momlife/main/content/plan.md
```
`status: 次` の中で最も日付が近い投稿を1件選ぶ。

**⚠️ 重要**：plan.mdはうりが直前に編集している可能性が常にある。実行直前に取得したファイルだけを正とし、それ以前にチャットで見た内容・記憶している内容は使わない。

### A-2: 時事ネタをチェック（30秒で判断）
教育・マネー・日常に関連する最新ニュースをWeb検索。Bloomberg / NYT / 公的統計を優先。関連があれば1スライドに差し込む。なければスキップ。

### A-3: カルーセル本文を作る
- 5〜8枚構成。1枚目フック / 中間1枚1メッセージ / 最終枚まとめ＋「保存してね🔖」CTA＋ブログ送客。
- 各データに出典を明記。

### A-4: Canva でテキスト差し替え
1. 対象テンプレートをコピー（`copy-design`）。
2. `start-editing-transaction` で開始。
3. `perform-editing-operations` で `find_and_replace_text` を使い全テキスト差し替え。
4. 必要なら背景写真を差し替え（`upload-asset-from-url` → `update_fill`）。
5. プレビュー確認 → `commit-editing-transaction` で保存。

### A-5: Slack に下書きを通知（確認依頼のみ・予約はまだしない）

**① #sns投稿確認**（`C0B8BC4BKV3`）— レビュー依頼

```
*タイトル*：[タイトル]
*Canva編集URL*：https://www.canva.com/d/[デザインID]

⏸️ *まだBlotatoには予約していません。* このスレッドに「OK」「投稿してOK」などの返信、または👍リアクションをいただいてから予約します。
✏️ 修正がある場合：Canvaで直接編集するか、このスレッドに返信してください。
```

**② #ig-caption**（`C0B9SRCC9R8`）— キャプション本文（コピペ用）

### A-6: Blotato に予約投稿を登録

**⚠️ 必須条件：A-5の確認を得てから実行する**

- 確認なしで時間切れ（投稿時刻が来る）になった場合は予約せず、Slackに「確認が取れなかったため本日の投稿は見送りました」と通知する。

**手順2（手順1が失敗した場合のフォールバック）**：

## B｜ブログ記事下書き（公開前日 JST 22:00 に実行）

> **ブログとIGの前後関係**：ブログはIGの前に出せると理想だが、間に合わない場合はIGを先に投稿し、ブログを後から公開してもよい（連動の順番は厳密でなくてよい）。

### B-1: planned_topics.md を読む
以下のURLから最新版を取得する：

### B-2: 事前質問を判断する（最大3つまで）
以下のいずれかに該当する場合、Slackで質問してから執筆に進む：
- うりの実体験の具体的な数字が必要（例：費用・期間・固有名詞）
- 身バレリスクの判断が必要（例：学校名・時期の特定）
- 公開範囲の確認が必要

質問する場合は `#sns投稿確認`（`C0B8BC4BKV3`）に以下の形式で投稿し、**その日は下書きを進めない**：
```
📋 *ブログ下書き前の確認事項 — [トピックタイトル]*

以下を教えてください（最大3点）：
1. [質問1]
2. [質問2（あれば）]
3. [質問3（あれば）]

回答いただいたら次回（火曜）に下書きを作成します。
```

### B-3: ファクトチェック・リサーチ
- トピックに関連する最新データをWeb検索（CRA公式・Statistics Canada・厚生労働省・権威報道など）
- 数値・制度名は必ず一次情報で確認。信頼できるソースがない場合は「現時点で確認できる信頼性のある情報は存在しません」と記し、推測で補完しない

### B-4: 下書きを書く（Cocoon HTML形式）

**⚠️ 最優先ルール：執筆前に必ず `blog/HTML_STYLE_GUIDE.md` を新規取得して読むこと。**
```
https://raw.githubusercontent.com/goraishared-design/jp.ca.momlife/main/blog/HTML_STYLE_GUIDE.md
```
このスタイルガイドのテンプレートを**コピーして埋める**形で記事を書く。新しい書式・ショートコード・クラス名を自分で考案してはいけない（過去に既存記事のデザインと揃わない下書きが生成された問題への対応）。

ガイドの特に重要な点（詳細は必ずガイド本体を参照）：
- 吹き出しは **Cocoonブロックコメント形式**（`wp:cocoon-blocks/balloon-ex-box-1`）。`[speech_bubble ...]` のようなショートコードは**存在しないので絶対に使わない**
- 著者名は必ず **Julie**（「うり」にしない）
- 全段落を `<!-- wp:paragraph -->` で囲む。見出しも `<!-- wp:heading -->` で囲む
- 表は**ネイビーヘッダー（#0d2240）＋ゼブラストライプ**のinline style形式。装飾なしの `<table>` は禁止
- 強調は `<span class="marker-under">` / `marker-under-red`（下線）を1〜2箇所に絞って使う
- タブボックス・blogcard・カード型図解もガイドの正式形式に従う
- 記事構成はガイド末尾「記事の標準テンプレート構成」の順番（リード文→Julie吹き出し→この記事でわかること→本編→まとめ→blogcard送客）で書く
- **記事冒頭に必ずメタ情報ヘッダー（タイトル・狙うキーワード・メタディスクリプション）の3点をHTMLコメントで書く**。メタディスクリプションは110〜120字で、主要キーワードを前半に含め、記事の結論や具体策を1つ入れる（ガイド「記事冒頭のメタ情報ヘッダー」参照）
- **毎回アフィリエイトを検討する**：ガイド「⑪アフィリエイトリンク管理」の保有リスト（Wise・保険のマンモス等）を確認し、記事テーマに自然に合うものがあればなるべく設置する。文脈に合わない場合は無理に貼らない。設置時はステマ表記と「最新は公式で確認」の注記を入れる

文字数の目安：3,000〜5,000字。確認できない情報は推測で埋めず「現時点で確認できる信頼性のある情報は存在しません」と書く。出典は本文または表の下に明記。

下書きは `blog/drafts/` フォルダに `.html` で保存する。

## トラブル時

- Canva の `replace_text` が "No approval received" → `find_and_replace_text` に切り替える。
- 画像URLが弾かれる → まずCanva `export-design` のURLを直接Blotatoの `mediaUrls` に渡す（手順1）。それでも失敗する場合のみ presigned upload 経由で永続URL化する（手順2）。両方失敗したらうりに手動アップロードを依頼。
- GitHub API が到達不可 → published_articles.md / planned_topics.md は直前のSlack通知から内容を把握し、後で同期。
- ファクトチェックで信頼できる情報が見つからない → 「現時点で確認できる信頼性のある情報は存在しません」と明記し、推測で補完しない。
- Threads投稿でエラー → ハッシュタグを5個以内に削減して再試行。
- plan.mdを編集した直後にRoutineが古い内容で動いてしまった（2026/6/13発生）→ 再発防止のためA-6・C-1に「うりの確認後に予約」ルールを追加済み。確認なしでBlotato予約は絶対に行わない。
