## 共通ルール（全タスクで守ること）

- **声・トーン**: Instagram カルーセルは知的で落ち着いた文体。Threads はカジュアルな話し言葉。ブログは実体験ベースの深い解説。
- **ファクトチェック必須**: 数値・固有名詞は必ず一次情報（公的機関・査読論文・権威報道）で確認し、出典を本文に明記。確認できない情報は使わない（推測で補完しない）。
- **身バレ防止**: 子どもの人数は「2人」で統一。早稲田はOK、学部は伏せる。帰国編入は「調べまくっている」視点（「実際に入学した」は使わない）。
- **3本柱**: ①教育（ディープグリーン）②マネー（ネイビー×ゴールド）③日常（テラコッタ×オフホワイト）。
- **Canva 自動生成は使わない**: 既存テンプレートのテキスト差し替えのみ。`find_and_replace_text` を使う（`replace_text` はブロックされる）。

## A｜IGカルーセル下書き（月・水・土の前日に実行）

### A-1: plan.md を読む
以下のURLから最新版を取得する：
```
https://raw.githubusercontent.com/goraishared-design/jp.ca.momlife/main/content/plan.md
```
`status: 次` の中で最も日付が近い投稿を1件選ぶ。

### A-2: 時事ネタをチェック（30秒で判断）
教育・マネー・日常に関連する最新ニュースをWeb検索。Bloomberg / NYT / 公的統計を優先。関連があれば1スライドに差し込む。なければスキップ。

### A-3: カルーセル本文を作る
- 5〜8枚構成。1枚目フック / 中間1枚1メッセージ / 最終枚まとめ＋「保存してね🔖」CTA＋ブログ送客。
- 各データに出典を明記。

### A-4: Canva でテキスト差し替え
1. 対象テンプレートをコピー（`copy-design`）。
2. `start-editing-transaction` で開始。
3. `perform-editing-operations` で `find_and_replace_text` を使い全テキスト差し替え。
4. 必要なら背景写真を差し替え（`upload-asset-from-url` → `update_fill`）。
5. プレビュー確認 → `commit-editing-transaction` で保存。

### A-5: Slack に下書きを通知
チャンネル `#sns投稿確認`（ID: `C0B8BC4BKV3`）に送る：
```
📋 *IGカルーセル下書き — [日付] [ピラー]*
*タイトル*：[タイトル]
*Canva編集URL*：https://www.canva.com/d/[デザインID]
*キャプション案*：[本文＋ハッシュタグ]

🕘 Blotatoに本日 JST 21:00 で予約登録しました。
✏️ 修正 → Canvaで直接 or このスレッドに返信
🚫 止めたい場合 → Blotatoダッシュボードでキャンセル
```

### A-6: Blotato に予約投稿を登録
- Canva `export-design` で各ページをエクスポート（一時URL取得）
- `blotato_create_presigned_upload_url` で永続URL化
- `blotato_create_post` で予約（Instagram account ID: `51161`、JST 21:00 = UTC 12:00）

## B｜ブログ記事下書き（火・木に実行）

### B-1: planned_topics.md を読む
以下のURLから最新版を取得する：
```
https://raw.githubusercontent.com/goraishared-design/jp.ca.momlife/main/blog/planned_topics.md
```
`status: 次` の中で `priority` が最も高いトピックを1件選ぶ。

### B-2: 事前質問を判断する（最大3つまで）
以下のいずれかに該当する場合、Slackで質問してから執筆に進む：
- うりの実体験の具体的な数字が必要（例：費用・期間・固有名詞）
- 身バレリスクの判断が必要（例：学校名・時期の特定）
- 公開範囲の確認が必要

質問する場合は `#sns投稿確認` に以下の形式で投稿し、**その日は下書きを進めない**：
```
📋 *ブログ下書き前の確認事項 — [トピックタイトル]*

以下を教えてください（最大3点）：
1. [質問1]
2. [質問2（あれば）]
3. [質問3（あれば）]

回答いただいたら次回（火 or 木）に下書きを作成します。
```

### B-3: ファクトチェック・リサーチ
- トピックに関連する最新データをWeb検索（CRA公式・Statistics Canada・厚生労働省・権威報道など）
- 数値・制度名は必ず一次情報で確認。信頼できるソースがない場合は「現時点で確認できる信頼性のある情報は存在しません」と記し、推測で補完しない

### B-4: 下書きを書く（Cocoon HTML形式）
WordPress / Cocoonテーマ用のHTMLブロック形式で出力。以下を積極的に活用：
- 吹き出し（うり：右・watery-green）でうりの実体験・感情を入れる
- タブボックス（tab-caption-box）で情報整理
- HTMLの表（wp:table or wp:html内）で図解・比較表
- アイコンリスト（iconlist-box）で箇条書き
- CTAボタンと関連記事ブログカードで送客動線

文字数の目安：3,000〜5,000字（SEO・読者の満足度を両立する長さ）

### B-5: Slack に下書きを通知
`#sns投稿確認` に以下の形式で送る：
```
📝 *ブログ下書き完成 — [タイトル]*

*狙うキーワード*：[主要キーワード3〜5個]
*連動IGカルーセル*：[日付と投稿テーマ]
*ファクトチェック出典*：[主な出典を列記]

⚠️ 確認事項（あれば）：[不明点・うりに判断してほしい箇所]

下書きHTMLファイルはGitHubの blog/drafts/ フォルダに保存しました。
WordPressにコピペで使えます。
```

### B-6: planned_topics.md を更新 ＋ 新トピック補充
1. 完成したトピックの `status` を `下書き済` に変更
2. `status: 次` が9件以下になった場合、新トピック案を1〜2件Slackに提案する：
   - 3本柱のバランスを見て不足しているピラーを優先
   - 直近のIGカルーセルと連動できるもの
   - SEOで競合が少ないニッチな実体験系を優先
3. うりが承認したら planned_topics.md に追記して10件を維持

## トラブル時

- Canva の `replace_text` が "No approval received" → `find_and_replace_text` に切り替える。
- 画像URLが弾かれる → Canva一時URLをBlotatoの presigned upload 経由で永続URL化する。
- GitHub API が到達不可 → planned_topics.md は直前のSlack通知から内容を把握し、後で同期。
- ファクトチェックで信頼できる情報が見つからない → 「現時点で確認できる信頼性のある情報は存在しません」と明記し、推測で補完しない。
