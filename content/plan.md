# 📅 コンテンツプラン — @jp.ca.momlife

> このファイルの使い方
> - **うり（人間）**：投稿のトピック・順番・切り口を変えたいときは、ここを GitHub の Web 画面で編集する。週1回（日曜）に翌週分を見直すのがおすすめ。
> - **Claude Routine（自動）**：実行時にこのファイルの最新版を読み、`status: 次` の中で最も日付が近い投稿の下書きを作る。
> - `status` の値：`次`（次に作る）/ `下書き済`（Slack通知済み・レビュー待ち）/ `予約済`（Blotato登録済み）/ `投稿済`

---

## 基本ルール（Routineが毎回守ること）

- **声**：Instagramカルーセルは知的で落ち着いた文体。Threadsはカジュアルな話し言葉（うりのX過去ツイートが基準）。
- **ファクトチェック必須**：数値・固有名詞は一次情報（公的機関・査読論文・権威報道）で確認し、出典をスライドに明記。確認できない情報は使わない。
- **時事ネタ**：毎回、教育・お金まわりの最新ニュース（Bloomberg / NYT / 公的統計など）を軽く検索し、関連があれば1スライド差し込む。無理に入れない。
- **身バレ防止**：子どもの数はSNS上「2人」。早稲田はOK、学部は伏せる。
- **Canva**：既存テンプレートのテキスト差し替えのみ（`find_and_replace_text` を使う。`replace_text` は不可）。自動デザイン生成はしない。
- **投稿時刻**：日本時間（JST）21時。Blotatoに予約登録。
- **送客**：カルーセル最終枚で関連ブログ記事へ誘導。

## 2週間ごとのレビュー（反応分析）

- Claudeが保存率・リーチ・反応の傾向を読み、次の2週間のプランに反映
- 早期はフォロワー数が少なくデータがノイズになりやすいので、数字に振り回されず傾向だけ見る
